vuepress 使用记录,记录一些使用 vuepress 的时候遇到的坑与一些 api,方便之后的使用

# 全局属性

    $page:获取到这个页面的信息 例如 path、和页面的frontMatter等属性
    $site:获取当前整个网站的信息 例如路由表
    $withBase: 可以拼接你的链接,帮你生成正确的路径,也可以在markdown里使用该语法

# 打包遇到的问题

因为 vuepress 是用 node 服务端渲染所以无法识别 window 和 dom 方法和属性,所以确保只在 beforeMount 或者 mounted 访问浏览器 / DOM 的 API;
展示一个对于 SSR 不怎么友好的组件可以使用 ClientOnly 组件包裹一下

<ClientOnly>
  <NonSSRFriendlyComponent/>
</ClientOnly>

# markdown 的配置

lineNumbers：是否在每个代码块显示行号 extractHeaders： Markdown 文件的 headers (标题 & 小标题) 会在准备阶段被提取出来 , 并存储在 this.$page.headers 中

    markdown: {
        lineNumbers: true, //是否在每个代码块显示行号
        extractHeaders: ['h2', 'h3', 'h4', 'h5', 'h6'], // 提取标题到侧边栏的级别，默认['h2', 'h3']
    },
